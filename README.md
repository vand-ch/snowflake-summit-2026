# Snowflake Summit 2026 - Smart Calendar

Interactive calendar for Snowflake Summit 2026 with smart conflict detection and schedule building.

🔒 **Private Repository** - Only accessible to Charlie Health organization members

## Features

- 📅 **Calendar View** - Browse all sessions by day with time slots
- ⭐ **My Schedule View** - See your selected sessions organized by day
- 🔍 **Search & Filter** - Find sessions by keyword, track, or selection status
- ⚠️ **Conflict Detection** - Automatically grays out conflicting time slots
- 📤 **Export/Import** - Save and share your schedule as CSV

## Usage

Visit the live site: [Summit Calendar](https://charliehealth.github.io/snowflake-summit-calendar-2026/)

**View Van's Schedule:** [Load with my_summit_schedule.csv](https://charliehealth.github.io/snowflake-summit-calendar-2026/?load=my_summit_schedule.csv)

Or download `index.html` and open it locally in your browser.

## Access

This site is hosted on GitHub Pages from a **private repository** within the CharlieHealth organization.

- ✅ Accessible to all Charlie Health GitHub organization members
- ✅ Requires GitHub authentication
- ❌ Not accessible to the public internet

## How to Use

1. Browse sessions in **Calendar View**
2. Click the ⭐ on sessions to add them to your schedule
3. Switch to **My Schedule** tab to see your selected sessions organized by day
4. Export your schedule as CSV to share with colleagues
5. Import someone else's CSV to see their schedule

## Sharing Your Schedule

### Option 1: Share a Direct Link (Internal Only)
Add your CSV to the repo and share a URL like:
```
https://charliehealth.github.io/snowflake-summit-calendar-2026/?load=my_summit_schedule.csv
```

This will automatically load your schedule when someone visits the link!

**Note:** Recipients must be logged into GitHub with access to the CharlieHealth organization.

### Option 2: Share the CSV File
Export your schedule and send the CSV file. Recipients can use the "Import Schedule" button to load it.

### Option 3: Share the HTML File
Send the `index.html` file directly - it's completely self-contained and works offline!

## Hosting

This is a static HTML file hosted on GitHub Pages from a private repository. No server or database required, but authentication through GitHub is enforced.
